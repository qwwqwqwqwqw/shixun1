# 人脸识别使用说明 — 如何注册新用户

> 负责人：C组员
> 最后更新：2026-07-13

## 概述

本系统支持通过人脸识别自动映射房间号并触发导航（酒店入住模拟功能）。
注册新用户需要两步操作：

1. **放置人脸照片** → `vision/face/known_faces/`
2. **更新映射表** → `dev_ws/src/guide_pkg/config/face_room_map.yaml`

---

## 注册步骤

### 步骤 1：准备人脸照片

- 格式：JPG 或 PNG
- 要求：
  - 正面照，五官清晰
  - 光线充足，面部无遮挡
  - 建议尺寸：300x300 ~ 800x800 像素
  - 画面中最好只有一张人脸

### 步骤 2：放置照片

将照片放入 `vision/face/known_faces/` 目录，文件名使用用户拼音或英文名：

```bash
# 示例：注册新用户 "赵六"
cp ~/Desktop/zhao_liu.jpg vision/face/known_faces/zhao_liu.jpg
```

> **文件名规则**：全小写拼音，下划线分隔，与映射表中的 key 完全一致。

### 步骤 3：更新映射表

编辑 `dev_ws/src/guide_pkg/config/face_room_map.yaml`，添加用户信息：

```yaml
face_room_map:
  zhang_san: "501"
  li_si: "502"
  wang_wu: "503"
  zhao_liu: "504"    # ← 新增
```

### 步骤 4：重新编译（如有修改）

```bash
cd dev_ws
colcon build --packages-select guide_pkg --symlink-install
source install/setup.bash
```

### 步骤 5：重启人脸识别节点

```bash
# 停止当前 face_recognizer 节点，重新运行
python3 vision/face/face_recognizer.py
```

节点启动后默认处于等待状态，不会持续占用摄像头。可通过小程序按钮启动，
或在 ROS2 终端执行：

```bash
ros2 topic pub --once /face_mode_control std_msgs/msg/String "{data: start}"
```

---

## 验证注册

1. 启动人脸识别模式后站到摄像头前。
2. 查看 `face_recognizer` 节点日志，应显示识别到的人名。
3. 检查识别过程状态：

```bash
ros2 topic echo /face_recognition_status
```

4. 检查 `/face_room` 话题是否发布了对应房间号：

```bash
ros2 topic echo /face_room
```

5. 确认小车开始导航到对应房间。

默认需要连续 2 次匹配才会确认，15 秒仍未识别则自动超时并释放摄像头。

---

## 删除用户

1. 删除 `vision/face/known_faces/` 下对应的照片文件
2. 从 `face_room_map.yaml` 中删除对应行
3. 重启人脸识别节点

---

## 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 无法识别 | 照片质量差 | 更换清晰正面照 |
| 识别错误 | 与其他用户相似 | 添加多张不同角度照片 |
| 映射失败 | 映射表未更新 | 检查 `face_room_map.yaml` |
| 节点报错 | 照片文件名与映射表 key 不一致 | 确保两者完全一致 |
| 提示未加载注册人脸 | `known_faces/` 中没有照片 | 添加 JPG/PNG 后重启节点 |
| 摄像头不可用 | 设备号变化或被其他节点占用 | `ls /dev/video*`，必要时用 `camera_index` 参数指定 |
| 识别超时 | 光线差、距离过远或未正对摄像头 | 改善光照并靠近摄像头后重试 |

## 隐私与数据安全

- 人脸照片属于敏感个人信息，采集前必须获得本人明确同意。
- 照片只保存在小车本地 `known_faces/`，不要提交到 Git 仓库。
- 不再使用的人员应及时删除照片和映射记录。
- 正式发布小程序前，需要在隐私保护指引中说明采集目的、保存位置和删除方式。

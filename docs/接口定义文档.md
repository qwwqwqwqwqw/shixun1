## 一、小程序 ↔ 服务端 TCP Socket 协议

### 连接方式

| 属性 | 值 |
|------|-----|
| 协议 | TCP Socket |
| 服务地址 | 小车 IP（如 `10.112.253.188`） |
| 端口 | `9090` |
| 编码 | UTF-8，每条消息以 `\n` 分隔（JSONL） |
| 小程序 API | `wx.createTCPSocket()` |

---

### 1. 小程序 → 服务端（请求）

#### 1.1 教室导航

```json
{"type":"navigate","room":"501"}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | ✅ | 固定 `"navigate"` |
| room | string | ✅ | 目标教室号，如 `"501"` |

#### 1.2 摇杆控制

```json
{"type":"joystick","vx":0.3,"vy":0.0,"wz":0.0}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | ✅ | 固定 `"joystick"` |
| vx | float | ✅ | 线速度 X，前进为正、后退为负，范围 -0.5~0.5 m/s |
| vy | float | ✅ | 线速度 Y，当前四向控制固定为 0 |
| wz | float | ✅ | 角速度 Z，左转为正、右转为负，范围 -1.5~1.5 rad/s |

小程序四向按钮使用以下指令：

```json
{"type":"joystick","vx":0.2,"vy":0,"wz":0}
{"type":"joystick","vx":-0.2,"vy":0,"wz":0}
{"type":"joystick","vx":0,"vy":0,"wz":0.6}
{"type":"joystick","vx":0,"vy":0,"wz":-0.6}
{"type":"joystick","vx":0,"vy":0,"wz":0}
```

依次表示：前进、后退、左转、右转、停止。

安全规则：

- 按住方向按钮时每 100ms 发送一次指令，松开立即发送零速度。
- 服务端限制 `vx/vy` 最大绝对值为 0.5，`wz` 最大绝对值为 1.5。
- `joystick_ctrl` 超过 0.5 秒未收到连续指令会自动发布零速度。
- TCP 断开、页面隐藏或退出时必须发送或补发零速度。
- 手动控制开始前先请求取消当前 Nav2 导航，避免两个节点同时控制底盘。

#### 1.3 取消导航

```json
{"type":"cancel"}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | ✅ | 固定 `"cancel"` |

#### 1.4 人脸识别模式

```json
{"type":"face_mode","action":"start"}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | ✅ | 固定 `"face_mode"` |
| action | string | ✅ | `"start"` 启动 / `"stop"` 停止 |

#### 1.5 心跳

```json
{"type":"ping"}
```

---

### 2. 服务端 → 小程序（响应/推送）

#### 2.1 指令确认 (ack)

```json
{"type":"ack","command":"navigate","room":"501","status":"accepted"}
```

#### 2.2 导航状态推送

```json
{"type":"nav_status","message":"开始导航到 501（来源: manual）"}
```

状态字符串值：`"开始导航到{room}（来源:{source}）"` / `"导航中..."` / `"到达{room}"` / `"导航失败:{reason}"`

#### 2.3 到达确认推送

```json
{"type":"arrival","message":true}
```

#### 2.4 心跳响应

```json
{"type":"pong","message":"ok"}
```

┌──────────┐   TCP:9090   ┌──────────────┐  /app_joystick  ┌─────────────┐  /cmd_vel  ┌──────────┐

## 二、ROS2 话题（Topics）

### 1. `/command_room` — 手动教室号指令

| 属性 | 值 |
|------|-----|
| 类型 | `std_msgs/String` |
| 发布者 | `aiserver_node`（D组员） |
| 订阅者 | `guide_node`（B组长） |
| 消息示例 | `{"data":"501"}` |

### 2. `/face_room` — 人脸识别房间号/控制指令

| 属性 | 值 |
|------|-----|
| 类型 | `std_msgs/String` |
| 发布者 | `face_recognizer`（C组员）/ `aiserver_node`（D组员） |
| 订阅者 | `guide_node`（B组长） |
| 说明 | `"501"` → 导航到房间；`"start"/"stop"` → 人脸模式开关 |

### 3. `/navigation_status` — 导航状态

| 属性 | 值 |
|------|-----|
| 类型 | `std_msgs/String` |
| 发布者 | `guide_node`（B组长） |
| 订阅者 | `aiserver_node`、`arrival_fusion` |

### 4. `/doorplate_result` — 门牌识别

| 属性 | 值 |
|------|-----|
| 类型 | `std_msgs/String` |
| 发布者 | `doorplate_detector`（C组员） |
| 订阅者 | `arrival_fusion`（B组长） |

### 5. `/arrival_confirmed` — 到达确认

| 属性 | 值 |
|------|-----|
| 类型 | `std_msgs/Bool` |
| 发布者 | `arrival_fusion`（B组长） |
| 订阅者 | `aiserver_node` |

### 6. `/app_joystick` — 小程序摇杆指令

| 属性 | 值 |
|------|-----|
| 类型 | `geometry_msgs/Twist` |
| 发布者 | `aiserver_node`（D组员，接收小程序摇杆 JSON） |
| 订阅者 | `joystick_ctrl`（B组长，安全转发） |
| 用途 | 小程序摇杆 → aiserver 转发 → joystick_ctrl 安全校验 → `/cmd_vel` |

### 7. `/cmd_vel` — 最终底盘速度

| 属性 | 值 |
|------|-----|
| 类型 | `geometry_msgs/Twist` |
| 发布者 | `joystick_ctrl` / Nav2 |
| 订阅者 | `icar_bringup` 底盘驱动 |
| 用途 | 底盘接收后驱动麦克纳姆轮运动 |

### 8. `/navigation_cancel` — 取消导航

| 属性 | 值 |
|------|-----|
| 类型 | `std_msgs/Bool` |
| 发布者 | `aiserver_node`（D组员） |
| 订阅者 | `guide_node`（B组长） |

## 四、数据流总图

```
小程序 ──TCP:9090──► aiserver_node
                      │
                      ├── /command_room ──► guide_node ──► Nav2 ──► /cmd_vel ──► 底盘
                      ├── /app_joystick ──► joystick_ctrl ──► /cmd_vel ──► 底盘（摇杆）
                      ├── /navigation_cancel ──► guide_node（取消导航）
                      ├── /face_room ──► guide_node（人脸导航/模式）
                      │
                      ◄── /navigation_status ── guide_node
                      ◄── /arrival_confirmed ── arrival_fusion

摄像头 ──► face_recognizer ── /face_room ──► guide_node
摄像头 ──► doorplate_detector ── /doorplate_result ──► arrival_fusion
```

## 五、QoS 策略

| 话题 | QoS | 说明 |
|------|-----|------|
| `/command_room` | RELIABLE, KEEP_LAST(10) | 指令不可丢失 |
| `/face_room` | RELIABLE, KEEP_LAST(10) | 指令不可丢失 |
| `/navigation_status` | RELIABLE, KEEP_LAST(10) | 状态实时反馈 |
| `/app_joystick` | BEST_EFFORT, KEEP_LAST(1) | 摇杆高频更新，允许丢帧 |
| `/cmd_vel` | BEST_EFFORT, KEEP_LAST(1) | 高频速度指令 |
| `/doorplate_result` | BEST_EFFORT, KEEP_LAST(1) | 视觉结果允许丢帧 |
| `/arrival_confirmed` | RELIABLE, KEEP_LAST(1) | 确认信号不可丢失 |

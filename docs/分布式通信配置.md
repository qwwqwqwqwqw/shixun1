# 分布式通信配置 — 跨容器/跨主机

> 负责人：B组长
> 最后更新：2025-07-11

## 概述

本项目支持两种分布式通信场景：
1. **跨容器通信**：Docker 容器间（nav 容器 + vision 容器）
2. **跨主机通信**：虚拟机（开发） ↔ 小车（运行）

ROS2 使用 DDS（Data Distribution Service）作为底层通信中间件，通过配置 DDS 参数实现分布式通信。

---

## 一、跨容器通信（Docker）

### 配置方案

Docker 容器间 ROS2 通信需要使用 `host` 网络模式。

`docker-compose.yml` 关键配置：
```yaml
services:
  ros2-nav:
    network_mode: host          # 使用宿主机网络
    environment:
      - ROS_DOMAIN_ID=42        # 所有容器必须一致
      - RMW_IMPLEMENTATION=rmw_fastrtps_cpp

  vision:
    network_mode: host
    environment:
      - ROS_DOMAIN_ID=42        # 与 nav 容器一致
      - RMW_IMPLEMENTATION=rmw_fastrtps_cpp
```

### 启动

```bash
# 启动所有容器
docker-compose up -d

# 查看容器状态
docker-compose ps

# 查看导航容器日志
docker logs -f icar-nav

# 查看视觉容器日志
docker logs -f icar-vision
```

### 验证跨容器通信

```bash
# 在 nav 容器中发布话题
docker exec -it icar-nav ros2 topic pub /command_room std_msgs/String '{data: "101"}'

# 在 vision 容器中监听
docker exec -it icar-vision ros2 topic echo /command_room
```

## 二、跨主机通信（虚拟机 ↔ 小车）

### 前提条件
- 虚拟机和小车在同一局域网
- 两端 ROS2 版本一致（Humble）
- 两端 DDS 实现一致（推荐 FastDDS）

### 配置步骤

#### 1. 设置相同的 ROS_DOMAIN_ID

**虚拟机：**
```bash
echo 'export ROS_DOMAIN_ID=42' >> ~/.bashrc
source ~/.bashrc
```

**小车（SSH 登录后）：**
```bash
echo 'export ROS_DOMAIN_ID=42' >> ~/.bashrc
source ~/.bashrc
```

> `ROS_DOMAIN_ID` 必须一致，否则无法互相发现节点。

#### 2. 设置 RMW 实现（推荐 FastDDS）

两端都执行：
```bash
echo 'export RMW_IMPLEMENTATION=rmw_fastrtps_cpp' >> ~/.bashrc
source ~/.bashrc
```

#### 3. 配置 FastDDS XML（可选，用于过滤网络接口）

如果有多块网卡，需要指定通信网卡。

创建 `~/fastdds_profile.xml`：
```xml
<?xml version="1.0" encoding="UTF-8" ?>
<profiles xmlns="http://www.eprosima.com/XMLSchemas/fastRTPS_Profiles">
  <transport_descriptors>
    <transport_descriptor>
      <transport_id>udp_transport</transport_id>
      <type>UDPv4</type>
    </transport_descriptor>
  </transport_descriptors>

  <participant profile_name="udp_profile" is_default_profile="true">
    <rtps>
      <userTransports>
        <transport_id>udp_transport</transport_id>
      </userTransports>
      <useBuiltinTransports>false</useBuiltinTransports>
      <builtin>
        <initialPeersList>
          <locator>
            <udpv4>
              <!-- 小车 IP -->
              <address>192.168.1.120</address>
            </udpv4>
          </locator>
        </initialPeersList>
      </builtin>
    </rtps>
  </participant>
</profiles>
```

两端都设置环境变量：
```bash
echo 'export FASTRTPS_DEFAULT_PROFILES_FILE=~/fastdds_profile.xml' >> ~/.bashrc
source ~/.bashrc
```

#### 4. 验证跨主机通信

**虚拟机上运行：**
```bash
ros2 topic pub /command_room std_msgs/String '{data: "101"}'
```

**小车上运行：**
```bash
ros2 topic echo /command_room
```

如果小车能看到虚拟机发布的消息，说明跨主机通信配置成功。

## 三、常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 节点互相看不到 | ROS_DOMAIN_ID 不一致 | 两端设置相同值 |
| 节点互相看不到 | 防火墙阻挡 | `sudo ufw allow` 放行 DDS 端口（7400-7500 UDP） |
| 通信延迟高 | 网络质量差 | 检查 WiFi 信号，优先使用有线 |
| Docker 容器看不到宿主机节点 | 未使用 host 网络 | `docker-compose.yml` 设置 `network_mode: host` |
| 多网卡环境无法发现 | DDS 绑定错误网卡 | 配置 FastDDS XML 指定网卡 |

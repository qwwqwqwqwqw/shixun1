# 最终测试报告

## 1. 项目概述
- 项目名称：icar_classroom_guide — 智能小车教室导航系统
- 核心功能：
  1. 小程序输入教室号控制导航
  2. 人脸识别自动映射房间号并导航（酒店入住模拟）
  3. 到达后门牌号视觉确认（YOLO+OCR，休眠-唤醒模式）
  4. 导航状态实时反馈（TCP 实时推送）
  5. 跨容器/跨主机分布式通信

```
小程序 ──TCP:9090──→ aiserver_node

                        ├── /command_room ──→ guide_node ──Nav2 Action──→ Nav2 导航栈
                        │                         │                        │
                        │                         ├── /initialpose ──→ AMCL
                        │                         │                        │
                        │                         │                  /amcl_pose ──→ 定位反馈
                        │                         │
                        │                         └── /navigation_status ──→ arrival_fusion
                        │                                                    │
                        ├── /app_joystick ──→ joystick_ctrl ── /cmd_vel ──→ 底盘
                        │
                        ├── /face_mode_control ──→ face_recognizer
                        │                              │
                        │                              └── /face_room ──→ guide_node
```

## 4. B组长（ROS核心节点）详细设计

### 4.1 guide_node — 核心导航节点

| 项目 | 说明 |
|------|------|
| 文件 | `dev_ws/src/guide_pkg/guide_pkg/guide_node.py` |
| 订阅 | `/command_room`, `/face_room`, `/navigation_cancel`, `/amcl_pose` |
| 发布 | `/navigation_status`, `/initialpose` |
| 动作客户端 | `/navigate_to_pose` (nav2_msgs/action/NavigateToPose) |
| 关键设计 | 双指令入口、导航中状态锁、AMCL定位自动初始化(最多3次重试)、自动选门、Nav2异步回调链 |

**导航流程**：
```
收到教室号 → 查 classrooms.yaml 坐标 → 检查导航中状态锁 → 检查定位状态
    → 自动选门(front/back) → 构造 PoseStamped → 等待 Nav2 服务就绪(3s超时)
    → send_goal_async → goal_response_callback → get_result_callback
    → 结果处理(4=到达/5=取消/6=中止) → 发布 /navigation_status
```

### 4.2 arrival_fusion — 多传感器融合节点

| 项目 | 说明 |
|------|------|
| 文件 | `dev_ws/src/guide_pkg/guide_pkg/arrival_fusion.py` |
| 订阅 | `/navigation_status`, `/doorplate_result` |
| 发布 | `/arrival_confirmed`, `/fusion_status` |
| 超时时间 | 50.0 秒 |
| 超时检查 | 定时器每秒检查 |

**融合逻辑**：
```
收到"到达 501" → 记录期望教室号，进入等待状态
    → 收到门牌识别结果"501" → 比对匹配 → 发布 arrival_confirmed=True
    → 收到门牌识别结果"502" → 比对不匹配 → 发布 fusion_status 提示
    → 50秒内未匹配 → 超时 → 发布 arrival_confirmed=False
```

### 4.3 joystick_ctrl — 摇杆/键盘控制节点

| 项目 | 说明 |
|------|------|
| 文件 | `dev_ws/src/guide_pkg/guide_pkg/joystick_ctrl.py` |
| 订阅 | `/app_joystick` |
| 发布 | `/cmd_vel` |
| 看门狗 | 0.5 秒自动停车 |
| 键盘键位 | 12键：u/i/o/j/k/l/m/,/. 全向移动，q/e 平移，w/x 加减速，空格急停 |
| 默认模式 | 小程序远程控制（keyboard_control=False） |

### 4.4 doorplate_detector — 门牌识别节点（B参与联调）

| 项目 | 说明 |
|------|------|
| 文件 | `vision/doorplate/doorplate_detector.py` |
| 识别方案 | YOLO 检测门牌区域 + Otsu 增强 + EasyOCR 读数字 |
| 工作模式 | **休眠-唤醒模式**：默认不占摄像头，到达后唤醒 |
| 唤醒信号 | 订阅 `/navigation_status`，收到 arrived 状态后激活 |
| 唤醒持续时间 | 20 秒（ACTIVE_DURATION） |
| 识别范围 | 3 位纯数字（含 OCR 混淆校正） |
| 帧处理间隔 | 1.0 秒 |
